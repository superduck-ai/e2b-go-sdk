# Billing & limits

E2B uses usage-based pricing. You pay for compute while sandboxes are running, and you can adjust template build resources when you need more CPU or memory.

## Customize build resources

When building templates from Go, set the resource limits in `BasicBuildOptions`.

```go
package main

import (
	"context"
	"log"

	e2b "github.com/superduck-ai/e2b-go-sdk"
)

func main() {
	template := e2b.Template(nil).FromBaseImage()

	_, err := e2b.Build(context.Background(), template, "my-template", &e2b.BuildOptions{
		BasicBuildOptions: e2b.BasicBuildOptions{
			CpuCount:    8,
			MemoryMB:    4096,
			OnBuildLogs: e2b.DefaultBuildLogger(),
		},
	})
	if err != nil {
		log.Fatal(err)
	}
}
```

## Cost guidance

- You are billed while a sandbox is actively running.
- Paused or killed sandboxes stop billing immediately.
- Start with default resources and scale up only when your workload needs it.
- Monitor running sandboxes and builds in the dashboard.

For plan details, billing management, spending controls, and current pricing, use the E2B dashboard and pricing pages.
