# SSH access

You can expose SSH from a sandbox by:

1. installing `openssh-server`
2. authorizing your public key inside the template
3. proxying WebSocket traffic to port `22` with `websocat`

The example below assumes your local public key file is available in the template context as `id_ed25519.pub`.

## Build the SSH template

```go
package main

import (
	"context"
	"log"

	e2b "github.com/superduck-ai/e2b-go-sdk"
)

func main() {
	template := e2b.Template(&e2b.TemplateOptions{
		FileContextPath: ".",
	}).
		FromUbuntuImage("25.04").
		AptInstall([]string{"openssh-server"}, nil).
		MakeDir("/home/user/.ssh", &struct{ Mode int }{Mode: 0o700}).
		Copy("id_ed25519.pub", "/home/user/.ssh/authorized_keys", &struct {
			User string
			Mode int
		}{
			User: "user",
			Mode: 0o600,
		}).
		RunCmd([]string{
			"sudo mkdir -p /run/sshd",
			"curl -fsSL -o /usr/local/bin/websocat https://github.com/vi/websocat/releases/latest/download/websocat.x86_64-unknown-linux-musl",
			"chmod a+x /usr/local/bin/websocat",
		}, &struct{ User string }{User: "root"}).
		SetStartCmd(
			"sudo /usr/sbin/sshd && sudo websocat -b --exit-on-eof ws-l:0.0.0.0:8081 tcp:127.0.0.1:22",
			e2b.WaitForPort(8081),
		)

	_, err := e2b.Build(context.Background(), template, "ssh-ready", &e2b.BuildOptions{
		BasicBuildOptions: e2b.BasicBuildOptions{
			CpuCount:    2,
			MemoryMB:    2048,
			OnBuildLogs: e2b.DefaultBuildLogger(),
		},
	})
	if err != nil {
		log.Fatal(err)
	}
}
```

## Create a sandbox from the template

```go
package main

import (
	"context"
	"log"

	e2b "github.com/superduck-ai/e2b-go-sdk"
)

func main() {
	sandbox, err := e2b.Create(context.Background(), "ssh-ready", nil)
	if err != nil {
		log.Fatal(err)
	}

	log.Println(sandbox.SandboxID)
}
```

## Connect from your machine

Install `websocat` locally, then connect through the sandbox host.

```bash
ssh -i ~/.ssh/id_ed25519 \
  -o 'ProxyCommand=websocat --binary -B 65536 - wss://8081-%h.e2b.app' \
  user@<sandbox-id>
```

## How it works

`websocat` accepts WebSocket traffic on sandbox port `8081` and forwards it to the local SSH daemon on port `22`. Because the sandbox host is port-addressable, your local SSH client can use `ProxyCommand` to tunnel through the WebSocket endpoint.
