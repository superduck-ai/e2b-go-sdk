# Connect to a Running Sandbox

Use `e2b.Connect(ctx, sandboxID, opts)` when you already know the sandbox ID and want to attach to an existing sandbox.

A common flow is:

1. list sandboxes with `e2b.List(...)`
2. pick a `SandboxID`
3. call `e2b.Connect(...)`

If you already have a `*e2b.Sandbox`, `sandbox.Connect(ctx, opts)` reconnects the same handle in place.

## Get the sandbox ID

Use `e2b.List(...)` to find a running sandbox and read its `SandboxID`.

```go
package main

import (
	"context"
	"log"

	e2b "github.com/superduck-ai/e2b-go-sdk"
)

func main() {
	ctx := context.Background()

	paginator := e2b.List(&e2b.SandboxListOpts{
		Query: &struct {
			Metadata map[string]string
			State    []e2b.SandboxState
		}{
			State: []e2b.SandboxState{e2b.SandboxState("running")},
		},
	})

	runningSandboxes, err := paginator.NextItems()
	if err != nil {
		log.Fatal(err)
	}
	if len(runningSandboxes) == 0 {
		log.Fatal("no running sandboxes found")
	}

	sandboxID := runningSandboxes[0].SandboxID
	log.Println(sandboxID)
}
```

## Connect to the sandbox

Once you have the ID, call `e2b.Connect(...)` and use the returned `*e2b.Sandbox` normally.

```go
package main

import (
	"context"
	"log"

	e2b "github.com/superduck-ai/e2b-go-sdk"
)

func main() {
	ctx := context.Background()

	sandbox, err := e2b.Connect(ctx, "sbx_123", nil)
	if err != nil {
		log.Fatal(err)
	}

	execution, err := sandbox.Commands.Run(ctx, "whoami", nil)
	if err != nil {
		log.Fatal(err)
	}

	result := execution.(*e2b.CommandResult)
	log.Printf("connected to %s as %q\n", sandbox.SandboxID, result.Stdout)
}
```

To control the reconnect timeout, pass `SandboxConnectOpts.TimeoutMs` in milliseconds:

```go
timeoutMs := 60_000
sandbox, err := e2b.Connect(ctx, "sbx_123", &e2b.SandboxConnectOpts{
	TimeoutMs: &timeoutMs,
})
```
