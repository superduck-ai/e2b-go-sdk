# Custom Templates

You can pre-pull MCP server images during template build to reduce startup time later.

In the Go SDK, this is done with:

- `FromTemplate("mcp-gateway")`
- `AddMcpServer(...)`

`AddMcpServer(...)` is only valid on the `mcp-gateway` base template.

## Build a template with MCP servers

```go
package main

import (
	"context"
	"log"

	"github.com/joho/godotenv"
	e2b "github.com/superduck-ai/e2b-go-sdk"
)

func main() {
	_ = godotenv.Load()

	template := e2b.Template(nil).
		FromTemplate("mcp-gateway").
		AddMcpServer("browserbase", []string{"exa"})

	buildInfo, err := e2b.Build(context.Background(), template, "my-mcp-gateway", &e2b.BuildOptions{
		BasicBuildOptions: e2b.BasicBuildOptions{
			CpuCount:    8,
			MemoryMB:    8192,
			OnBuildLogs: e2b.DefaultBuildLogger(),
		},
	})
	if err != nil {
		log.Fatal(err)
	}

	log.Println(buildInfo.TemplateID, buildInfo.BuildID)
}
```

You can also pass several server names directly:

```go
template := e2b.Template(nil).
	FromTemplate("mcp-gateway").
	AddMcpServer("browserbase", "exa", "notion")
```

## Use the template at runtime

Even with a pre-pulled template, you still pass the runtime configuration through `SandboxOpts.Mcp`.

```go
package main

import (
	"context"
	"log"

	e2b "github.com/superduck-ai/e2b-go-sdk"
)

func main() {
	ctx := context.Background()

	sandbox, err := e2b.Create(ctx, "my-mcp-gateway", &e2b.SandboxOpts{
		Mcp: e2b.McpServer{
			"browserbase": map[string]any{
				"apiKey":       "browserbase-api-key",
				"geminiApiKey": "gemini-api-key",
				"projectId":    "project-id",
			},
			"exa": map[string]any{
				"apiKey": "exa-api-key",
			},
		},
	})
	if err != nil {
		log.Fatal(err)
	}
	defer sandbox.Kill(context.Background(), nil)
}
```
