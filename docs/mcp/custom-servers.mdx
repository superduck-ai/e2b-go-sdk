# Custom Servers

In the Go SDK, MCP runtime configuration is passed through `SandboxOpts.Mcp`, which is an `e2b.McpServer` alias for `map[string]any`.

That means custom MCP server definitions are passed through directly as Go maps.

## Configure a custom MCP server

```go
package main

import (
	"context"
	"log"

	e2b "github.com/superduck-ai/e2b-go-sdk"
)

func main() {
	ctx := context.Background()

	sandbox, err := e2b.Create(ctx, "", &e2b.SandboxOpts{
		Mcp: e2b.McpServer{
			"github/modelcontextprotocol/servers": map[string]any{
				"installCmd": "npm install",
				"runCmd":     "sudo npx -y @modelcontextprotocol/server-filesystem /root",
			},
		},
	})
	if err != nil {
		log.Fatal(err)
	}
	defer sandbox.Kill(context.Background(), nil)

	mcpURL := sandbox.GetMcpUrl()
	mcpToken, tokenErr := sandbox.GetMcpToken()

	_ = mcpURL
	_ = mcpToken
	_ = tokenErr
}
```

## Configuration shape

For a custom server entry, the E2B gateway expects fields such as:

- `installCmd`
- `runCmd`

Because the current Go SDK passes MCP config through as `map[string]any`, these keys are not modeled as dedicated Go structs.

## Notes

- `runCmd` should start an MCP server that communicates over stdio.
- If the server depends on `npx` or local packages, include an `installCmd` first.
- Since the config is passed through directly, keep the key names aligned with the gateway’s expected JSON fields.
