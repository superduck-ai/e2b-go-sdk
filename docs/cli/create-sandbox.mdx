# Create sandbox

The Go SDK equivalent to `e2b sandbox create <template>` is `e2b.Create(ctx, template, opts)`.

If you also want an interactive shell, create a PTY after the sandbox exists.

```go
package main

import (
	"context"
	"log"

	e2b "github.com/superduck-ai/e2b-go-sdk"
)

func main() {
	ctx := context.Background()
	ptyTimeoutMs := 0

	sandbox, err := e2b.Create(ctx, "base", nil)
	if err != nil {
		log.Fatal(err)
	}
	defer sandbox.Kill(context.Background(), nil)

	terminal, err := sandbox.Pty.Create(ctx, &e2b.PtyCreateOpts{
		Cols:      120,
		Rows:      36,
		TimeoutMs: &ptyTimeoutMs,
		OnData: func(data e2b.PtyOutput) {
			log.Print(string(data))
		},
	})
	if err != nil {
		log.Fatal(err)
	}
	defer terminal.Kill()

	if err := sandbox.Pty.SendInput(ctx, terminal.Pid, []byte("whoami\n"), nil); err != nil {
		log.Fatal(err)
	}
}
```

The `defer sandbox.Kill(...)` line gives you CLI-like behavior where the sandbox is torn down when your local session ends. Remove it if you want the sandbox to stay alive.
